## SSE + 进程池 + 限流 CR 修复
CR 发现并修复 4 个问题：
1. **P0 `versions.py` 进程池检查时序错误** — 先停旧 agent + commit DB 再检查上限，超限时旧 agent 已停、DB 已标 active 但没有进程在跑。改为：检查在停旧 agent 之前，超限时回滚 DB
2. **P1 `manager.py` SSE 空行被过滤** — `if line:` 丢掉了 SSE 事件分隔符（空行），导致不符合 SSE 协议。移除过滤
3. **P1 `miao_runner.py` sync generator 阻塞事件循环** — `for chunk in result` 是同步迭代，阻塞 uvicorn。加 `await asyncio.sleep(0)` 让出事件循环
4. **P2 令牌桶非线程安全** — `try_acquire_token()` 在线程池中执行但无锁。加 `threading.Lock` 保护；`__post_init__` 改为无条件初始化
5. **P2 `get_event_loop()` 已弃用** — 改为 `get_running_loop()`

## Docker 沙箱 + Webhook 异步回调 CR

发现 18 个问题（3 P0 / 10 P1 / 5 P2），产出 `docs/CR_DOCKER_WEBHOOK.md`：

**P0 Critical**:
1. Docker 模式 invoke/invoke_stream 检查 `self.process` 恒为 None → 所有 Docker agent 调用必崩
2. Dockerfile ENV 把 LANGFUSE_SECRET_KEY 烘进镜像层 → `docker history` 可见密钥
3. `COPY . /app/` 把 .venv/source.zip 等垃圾全塞进镜像

**P1 Important**:
4. image_exists() 只看 tag 不看 content hash → 代码更新后可能用旧镜像
5. Docker 容器名确定性 → 旧容器残留时 `docker run` 报冲突
6. try_restart() 不支持 Docker 模式 → Watchdog 无法重启 Docker agent
7. invoke_async 中 agent_id 查询冗余混乱含死变量
8. get_async_task_status 不验证 agent_name 归属 → 信息泄露
9. InvokeTask.created_at 用 datetime.utcnow() 已弃用 + naive datetime
10. TaskWorker asyncio.set_event_loop 不必要
11. webhook 在 DB commit 后发送 → 状态不一致窗口
12. retry_count/max_retries/next_retry_at DB 字段从未使用（死 schema）

**P2 Minor**: dict 线程安全缺注释、request_id 碰撞、相对 URL、端口硬编码、docker info 性能
