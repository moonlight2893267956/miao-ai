## Watchdog 空闲停止 + SSE 无输出修复
- **问题1**：watchdog idle_stop 300s 后停掉 agent 并从 registry 移除 → 后续 invoke 返回 503
  - 修复：idle_stop 后保留在 registry 中（不 remove），invoke 时自动唤醒
  - 修复：invoke.py 新增 `_try_auto_activate()` — 检测到 stopped agent 时自动 build_and_start
  - 修复：前端 Run 按钮不再限制 agent.status === "running"
- **问题2**：SSE 流式模式下非 generator agent 无 output 显示
  - 根因：miao_runner 的 `/invoke/stream` 对非 generator 结果不发 output 事件，只在 done 中返回 trace_id
  - 修复：miao_runner 非 generator 分支新增 `event: output` 事件，包含完整返回值
  - 修复：前端 SSE 解析新增 `output` 事件处理，直接显示 JSON 格式化结果

## 真正的逐 token 流式输出
- **根因**：sample-agent 的 `invoke()` 返回普通 dict，不是 generator，SSE 只有一个 output+done 事件
- **修复**：
  - `demos/sample-agent/agent.py` 新增流式模式：`config.get("stream")` 时用 `_chain.stream()` 逐 token yield
  - `backend/app/api/invoke.py` invoke_stream 端点自动注入 `config["stream"] = True`
  - `backend/agent_templates/miao_runner.py` 改用 `inspect.isgenerator()` 精确检测 generator
  - `frontend` token 事件提取 `token` 字段渲染文本，加打字光标动画，`whitespace-pre-wrap` 保留换行
  - `frontend` 非 stream invoke 也提取 `answer` 字段显示纯文本
- 上传 v3 版本并激活，curl 测试确认逐 token SSE 正常
