## 前端"挂掉"溯源 + status.sh 改进
- 用户反馈前端"挂掉"，`status.sh` 报"未运行"
- 排查发现：当前 PID 98816 (pnpm dev) 启动 8 分钟前（= 23:27），是被我自己前面的 nohup 启动的，**实际进程很健康**（HTTP 200 58ms）
- 真正的"老"进程已死，**无 crash report、无 log 残留**（被 nohup `>` 重定向 truncate 了）
- 最可能根因：**shell session 关闭时整个 process group 被 SIGKILL** —— 98816 现在 PPID=1（被 launchd 接管）证实了原 session 已退出
- macOS `pmset` 显示 `SleepDisabled=0`，系统睡眠也可能加重
- **无法 100% 确认**，但 session 关闭回收进程组是最常见且吻合证据的原因
- 改进 `scripts/status.sh`：
  - 新增端口 3000 监听检测（`lsof -nP -iTCP:3000 -sTCP:LISTEN`）
  - 区分三种状态：PID 文件存活 + 端口在监（健康）/ 仅端口在监（孤儿进程警告）/ 都没了（未运行）
  - 避免"进程活着但 PID 文件 stale 被误判"的情况
- 后续防护建议（未实施，等用户确认）：start-all.sh 用 `setsid` 而非 `nohup`，或加 launchd plist 持久化

## sample-agent v4 — 修复 generator 污染 return 的问题
- v3 用了 `if config.get("stream"): yield ... else: return ...` — Python 会把整个函数当 generator，`return` 值被丢弃
- 修复：把流式逻辑包进内部 `_stream()` 函数，外层 invoke 走 `return _stream()` 或 `return {...}`
- 重新打包上传 v4 并激活，stream + 非 stream 两种模式均正常
